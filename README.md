# happycat

## 概述  
1. 该工程包含两个核心模块：Android资源索引系统和跨平台社交内容分享解决方案，分别类似于移动开发的"资源字典"和"社交发布中枢"。  
2. 资源系统专注统一管理应用素材（如颜色、布局），而分享模块实现多平台内容分发（如图文编辑、跨平台发布），两者共同服务于移动应用的高效开发和社交化功能集成。  
3. 关键技术特征包括分层资源ID分类（类似文件目录）、智能图像三级缓存策略（内存/磁盘/网络）、以及混合交互模式（同步绑定+异步回调），依赖Android基础框架及第三方SDK如高德定位。  

## 什么是happycat?  
1. **核心模块交互**：资源系统通过Java常量（如`R.color.primary`）为上层提供素材索引，分享模块则利用这些资源构建UI，同时通过平台适配器（如PlatformGridViewAdapter）连接各大社交平台。图像加载框架作为桥梁，统一处理本地与网络图片的加载需求。  

2. **关键技术实现**：  
   - 资源ID采用编译时静态绑定（类似电话区号分配），由AAPT工具自动生成整型常量；  
   - 图像加载通过SmartImageTask实现"取件-处理-派送"流程，结合Handler机制完成线程间通信；  
   - 电商模块采用Fragment集合作为"可插拔页面容器"，通过Gson和HttpUtils实现数据流动。  

3. **典型应用场景**：  
   - 开发阶段：Android Studio通过资源ID提示加速布局编写（如`@string/app_name`）；  
   - 运行时：用户摇一摇手机触发分享模块，图像框架自动从内存缓存加载头像；  
   - 电商场景：RefreshListView下拉刷新时，混合交互模式确保数据实时性与流畅体验。
